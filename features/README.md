# features

> One directory per hardware feature вЂ” KiCad projects, schematics, PCB layouts, analysis, and documentation all live here.

---

## What is this directory?

Each subdirectory in `features/` represents a single hardware feature. Features are created using the `init/<feature>` PR process and scaffolded automatically by CI вЂ” do not create feature directories manually.

---

## Creating a new feature

Do not create a feature directory by hand. Instead, follow the init PR process:

1. Create a branch `init/<feature>`.
2. Push the branch so `init-branch-setup.yml` scaffolds the full directory structure automatically.
3. Pull the scaffold commit, add the required seed files, then open and merge the PR.

`init-feature.yml` is currently disabled. The scaffold is committed to the `init/<feature>` branch when it is first pushed, before the PR is opened.

See [docs/how-to/init-feature.md](../docs/how-to/init-feature.md) for step-by-step instructions.

## The one-feature rule

> вљ пёЏ Warning: A single PR must only touch KiCad files (`.kicad_sch`, `.kicad_pcb`, `.kicad_pro`) in **one** feature directory. CI (`pr-checks.yml`) will reject any PR that modifies files in multiple feature directories.

> вљ пёЏ Warning: Never manually resolve conflicts in `.kicad_sch` or `.kicad_pcb` files. KiCad file format is not merge-safe вЂ” conflicts will produce silently corrupted files.

## KiCad project files

The KiCad project files for a feature live at the root of the feature directory:

```
features/<feature>/<feature>.kicad_pro
features/<feature>/<feature>.kicad_sch   в†ђ top-level schematic
features/<feature>/pcb/<feature>.kicad_pcb
```

Hierarchical sub-sheets live in `schematics/` and are automatically discovered by the visual diff and render tools.
